# Contributing to BankStatementParsing

Thank you for your interest in contributing to BankStatementParsing!

## How to Contribute

- Fork the repository and create your branch from `main` or the latest development branch.
- Follow .NET and C# coding conventions and best practices.
- Write clear, concise commit messages.
- Add or update tests as appropriate for your changes.
- Ensure all tests pass before submitting a pull request.
- Document new features or changes in the relevant markdown files.

## Pull Request Process

1. Ensure your code builds and passes all tests.
2. Open a pull request with a clear description of your changes.
3. Reference any related issues in your PR description.
4. One of the maintainers will review your PR and provide feedback or merge it.

## Code of Conduct

Please be respectful and considerate in all interactions. See [Contributor Covenant](https://www.contributor-covenant.org/) for guidelines.

## Reporting Issues

- Use the GitHub Issues page to report bugs or request features.
- Provide as much detail as possible, including steps to reproduce, logs, and screenshots if applicable.

## Questions?

Open an issue or start a discussion on GitHub.
